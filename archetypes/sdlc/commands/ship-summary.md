description: Summarize completed SDLC work for handoff or PR prep.
---

Use the completed SDLC artifacts to produce a concise `ship-summary`.

Treat the user message that follows this prompt as the final `specification`, `implementation-plan`, `execution-report`, and `review-report`.

Summarize:

1. what was requested
2. what was implemented
3. what changed in the codebase
4. tests and validation that were run
5. review outcome
6. approved follow-ups, known risks, or deferred items

Keep the summary concise and suitable for a PR description, release note draft, or stakeholder handoff.
