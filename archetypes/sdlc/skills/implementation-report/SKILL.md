# Implementation Report Skill

Produce an `execution-report` with these sections:

## Completed plan items

Map each completed item to the relevant step in the `implementation-plan`.

## Changed areas

Group changes by logical area rather than by file list alone.

## Tests and verification

State what was added, updated, or run and what confidence that provides.

## Deviations

Document every deviation from the plan and explain why it was necessary.

## Remaining risks

State residual concerns, follow-ups, or deliberately deferred work.
